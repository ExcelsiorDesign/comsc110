Course Description
Introduction to programming emphasizing modular design and development of programs, coding style, documentation, debugging and testing. All control structures and data types of a commonly used language are covered. 

Course Content

  • Review of computer and programming terminology 
  • Data types; constants, variables expressions and assignment statements 
  • Input/output 
  • Subprograms (C++ functions or Java methods) 
  • Structure charts, flowcharting, algorithms, and pseudo code 
  • Conditions and logical operators, IF statements, and loops 
  • Data representation and advanced data types 
  • Structured types, one-dimensional arrays and multidimensional arrays 
  • File (Text files) basics 
  • Simple recursive functions. 
  • Dynamic data structures and singly linked lists 
  • Collections

Student Learning Objectives - Students will be able to:

  1. Analyze problem specifications;
  2. Use variables, branching, looping, libraries, and subprograms;
  3. Design, code, debug, test, and document programs written to solve diverse problems;
  4. Apply the fundamentals of good programming structure and good programming practices;
  5. Use data structures such as records, files, and arrays, and to assess their suitability for various applications;
  6. Use C++ pointers or Java reference variables.
  7. Use recursion.

Submission of Assignments:
Assignment report must be submitted on time, i.e. before the posted due time on the CANVAS calendar. Please be conscience about the due time and give yourself enough time to work on the homework problems.
Please submit ONE and only one all-inclusive report file per assignment. The homework report must contain both the source code and validation rest run console output. The preferred report format would be the Online Entry type.

The source code copied and pasted directly on text  and a screen shot of one or more runs included in the same word document where the source code is. You can use open office (free) if you do not have MS word.  

Please note that assignments submitted any other will not be accepted for grading.

Incomplete 
The “incomplete” grade was designed for students who have a last minute “disastrous” event (serious illness, death in the family) that prevents them from completing the course. A student must be passing prior to the “disaster”, must have more than 50 percent of the work completed, and evidence of the “disaster” must be supplied to the instructor. The instructor has sole discretion of whether to issue the incomplete. Incompletes are rarely given and do require a contract between the student and instructor about what work is to be made up and the time limit in which it is to be finished.

Student Privacy Rights Policy
The Family Educational Rights and Privacy Act (FERPA) is a law that protects students from having their education records shared with people without the student giving their advance approval in writing.  (Education records include such things as grades, information from counseling appointments, academic standing, disciplinary records, to name a few.)  FERPA also gives students the right to see their education records.  If there is inaccurate or misleading information in their records, students can challenge it and try to get it changed.  Under FERPA, colleges must have students’ written permission to let parents, guardians, spouses, or others see or discuss their educational records, even if the student is present.
